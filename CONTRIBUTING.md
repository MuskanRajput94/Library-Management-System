# Contributing

Thanks for your interest in contributing to this project!

## How to Contribute

1. **Fork** the repository
2. **Clone** your fork locally
3. Create a new **branch** for your feature or fix
4. Make your changes and **test** them
5. **Commit** with a clear message
6. **Push** to your fork and open a **Pull Request**

## Guidelines

- Keep changes focused — one feature or fix per PR
- Follow the existing code style (vanilla JS, no frameworks)
- Test your changes locally before submitting
- Update the README if your change affects setup or usage

## Reporting Bugs

Open an [issue](https://github.com/MuskanRajput94/Library-Management-System/issues) with:
- What you expected to happen
- What actually happened
- Steps to reproduce
- Browser/OS info if relevant

## Suggestions

Feature ideas and improvements are welcome — open an issue to discuss before starting work on large changes.
